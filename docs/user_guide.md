# UCSK User Guide

## What is UCSK?

UCSK (Unified Cognitive Singularity Kernel) is a living digital entity — a
fully integrated engineering ecosystem that combines multimodal sensing,
intelligent agents, and adaptive interfaces to evolve with you as a
programmer.

The system:
- **Senses** your mental state via webcam, microphone, and behaviour
- **Adapts** the interface to match your cognitive state
- **Assists** via 13 coordinated AI agents
- **Remembers** every interaction for long-term evolution
- **Shares** anonymised skills across a P2P network

---

## Getting Started

### 1. Launch the Stack

```bash
cd infrastructure
docker compose up --build
```

This starts:
- **Backend** at http://localhost:8000 (FastAPI)
- **Frontend** at http://localhost:5173 (Vite + React)
- **Qdrant** at http://localhost:6333
- **Neo4j** at http://localhost:7474
- **PostgreSQL** at localhost:5432
- **Redis** at localhost:6379

### 2. Open the UI

Navigate to http://localhost:5173. The UI starts in **Visual** mode with a
Monaco code editor and agent suggestion panel.

### 3. Grant Camera/Mic Access (Optional)

For full cognitive sensing, allow camera and microphone access when prompted.
The system works without them but relies solely on keyboard/mouse behaviour.

## UI Modes

UCSK automatically switches between five interface modes:

| Mode | When | What You See |
|------|------|-------------|
| **Visual** | Focused | Full editor + agent suggestions + Three.js visualisation |
| **Audio** | Fatigued / Frustrated | Calm screen with voice interaction |
| **Haptic** | Touch devices | Large tactile cards with haptic feedback |
| **Mixed** | Distracted | Editor + voice-guided suggestions |
| **Zero** | Flow / Idle | Nearly invisible — agents run silently |

Mode switches include a 2-second cooldown to prevent flicker.

---

## Mental States

The system classifies your cognitive state into:

| State | Indicators |
|-------|-----------|
| **Focused** | Steady typing, good eye contact, moderate blink rate |
| **Flow** | High engagement + focus, minimal errors |
| **Distracted** | Low typing speed, high mouse jitter, gaze wander |
| **Fatigued** | High blink rate, long idle gaps, low energy |
| **Frustrated** | High error rate, fast speech, negative expression |
| **Idle** | No activity detected |

## Agent Mesh

### 5 Supervisors

1. **Sensory** — aggregates raw sensing data
2. **Analysis** — interprets code context and recent actions
3. **Interface** — coordinates UI mode transitions
4. **Execution** — dispatches tasks to specialists
5. **Memory** — persists snapshots and recalls relevant history

### 8 Specialists

1. **Backend** — server-side code analysis
2. **Low-Level** — systems programming, performance
3. **Security** — credential scanning, vulnerability detection
4. **Research** — arXiv/GitHub monitoring
5. **Code Review** — style enforcement, anti-pattern detection
6. **Testing** — test generation, coverage analysis
7. **Architecture** — design review, coupling metrics
8. **DevOps** — CI/CD, deployment management

## Skill Diffs & Federated Learning

**Skill Diffs** are anonymised knowledge deltas generated by Gemini:
- Your demonstrated expertise is converted into abstract exercises
- No raw code or personal data is ever shared
- Other users receive tailored practice prompts

The P2P network uses WebRTC data channels for decentralised skill transfer.

---

## Privacy & Security

- All cognitive processing runs locally
- Facial data is processed in real-time and never stored as images
- User IDs are one-way hashed before any network communication
- The Security Agent continuously scans for credential exposure
- The Ethical Constitution (Feature 14) prevents privacy violations

## Configuration

All settings are controlled via environment variables (prefix: `UCSK_`):

| Variable | Default | Description |
|----------|---------|-------------|
| `UCSK_GEMINI_API_KEY` | (required) | Google Gemini API key |
| `UCSK_QDRANT_HOST` | localhost | Qdrant server host |
| `UCSK_NEO4J_URI` | bolt://localhost:7687 | Neo4j connection URI |
| `UCSK_POSTGRES_DSN` | postgresql+asyncpg://ucsk:ucsk@localhost:5432/ucsk | PostgreSQL DSN |
| `UCSK_REDIS_URL` | redis://localhost:6379/0 | Redis URL |
| `UCSK_DEBUG` | false | Enable debug logging |

Create a `.env` file in the project root for local development.

## Troubleshooting

| Issue | Solution |
|-------|---------|
| Camera not detected | Check browser permissions; ensure HTTPS or localhost |
| Whisper loading slow | First run downloads the model (~140 MB for "base") |
| Neo4j connection refused | Ensure `docker compose up neo4j` is running |
| Frontend WebSocket error | Check that backend is running on port 8000 |
| Skill Diff empty | Set `UCSK_GEMINI_API_KEY` environment variable |
